# Walk-Forward Validation

走前驗證（Walk-Forward Analysis）方法論說明。

## 概念

Walk-Forward 驗證是一種嚴謹的時序回測方法：在訓練期內找到最佳參數後，立即在隨後的測試期驗證，確保不使用未來資料。

```
時間軸 →
┌──────────────┬──────┐
│   訓練期      │ 測試 │  第 1 窗口
└──────────────┴──────┘
      ┌──────────────┬──────┐
      │   訓練期      │ 測試 │  第 2 窗口（滾動）
      └──────────────┴──────┘
            ┌──────────────┬──────┐
            │   訓練期      │ 測試 │  第 3 窗口
            └──────────────┴──────┘
```

## 本平台的實作

本平台的回測（`backtest.py`）採用「純樣本外測試」設計：

1. **策略固定**：評分模型參數（權重、門檻）在分析前已由 `config.py` 設定。
2. **推薦日確定**：每日收盤後執行分析，推薦名單寫入 DB（帶有日期戳）。
3. **報酬計算**：用推薦日之後的真實收盤價計算持有報酬。
4. **沒有回頭看**：`_ret_at()` 函數從推薦日往後數交易日，不使用推薦日之前的「未來價格」。

這等同於每個推薦都是一個獨立的「樣本外」測試點。

## 滾動累積驗證

Dashboard 回測頁的「月報酬趨勢」圖顯示：

- **X 軸**：時間（推薦月份）
- **Y 軸**：當月所有推薦的平均報酬率（20 日持有）
- **顏色**：模型（紫色）vs 0050 基準（灰色）

這讓使用者觀察模型在不同市場環境下的表現一致性。

## 模型漂移監控

信心評分（★★★★★）整合了以下信號：
- **樣本積累**（n）：樣本越多，統計越可靠
- **p-value**：t 檢定顯著性隨時間更新
- **Alpha 穩定性**：IR（Information Ratio）衡量 Alpha 的一致性

當 Alpha 從正轉負，或 p-value 上升到 0.20 以上，信心評分自動下降，提示策略可能出現漂移。

## 操作建議

| 樣本數 | 建議 |
|---|---|
| < 20 | 不建議解讀，繼續蒐集資料 |
| 20–40 | 初步參考，避免過度最佳化 |
| 40–80 | 可進行參數微調 |
| > 80 | 具備一定統計可靠性 |

**如果 Alpha 統計不顯著（p > 0.10）**：
- 不要修改評分參數以「套配」歷史資料（避免過擬合）
- 先增加觀察窗口，確認是系統性問題還是特定市場環境造成
