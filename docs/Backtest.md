# Backtest

回測驗證方法論。程式碼位於 `dashboard/pages/backtest.py`。

## 設計原則

- **不用未來資料**：所有分析以推薦當日的資料為基準，報酬率計算使用推薦日之後的實際收盤價。
- **交易成本**：每筆來回（買+賣）扣除 `ROUND_TRIP_COST = 0.585%`（含券商手續費 0.285% × 2 + 證交稅 0.3% 單邊 × 0.5 折算）。
- **基準**：0050（市值加權）和 0056（高股息）的對應持有期報酬，計算 Alpha。

## 持有期

| 持有期 | 說明 |
|---|---|
| 20 個交易日（約 1 個月）| 短期動能驗證 |
| 60 個交易日（約 3 個月）| 中期品質驗證 |

## 計算流程

```
推薦名單（歷史）
  │
  ├─ 找推薦日後第 N 個交易日的收盤價
  │     _ret_at(plist, ref_date, n_trading_days)
  │
  ├─ 計算報酬率（扣除交易成本）
  │     ret = (exit_price - entry_price) / entry_price × 100 - ROUND_TRIP_COST
  │
  ├─ 計算基準報酬（0050 / 0056 同期持有）
  │
  └─ 計算 Alpha = 個股報酬 - 基準報酬
```

## 統計指標

`_calc_stats(ret, alpha, hold_days)` 回傳：

| 欄位 | 說明 |
|---|---|
| `n` | 樣本數 |
| `mean_ret` | 平均報酬率（%）|
| `mean_alpha` | 平均超額報酬（Alpha，%）|
| `win_alpha` | Alpha > 0 的比率（%）|
| `ir` | Information Ratio = α / σ(α) |
| `sharpe` | 年化 Sharpe Ratio |
| `t`, `p` | t 統計量與 p-value（單樣本 t 檢定：H₀：Alpha = 0）|
| `sig` | p < 0.05（統計顯著）|
| `mdd` | 最大回撤（%）|

## 模型信心評分

`_model_confidence(stats)` 依下列四個維度各給 0–3 分，合計 0–12 分，映射到 1–5 星：

| 維度 | 3 分 | 2 分 | 1 分 | 0 分 |
|---|---|---|---|---|
| 樣本數 | ≥ 80 | ≥ 40 | ≥ 20 | < 20 |
| p-value | < 0.05 | < 0.10 | < 0.20 | ≥ 0.20 |
| Alpha | > 2% | > 0.5% | > 0% | ≤ 0% |
| IR | > 0.5 | > 0.2 | > 0 | ≤ 0 |

評語說明：
- ★★★★★ — 優秀，Alpha 統計顯著
- ★★★★ — 良好，具初步超額報酬能力
- ★★★ — 尚可，需跨越更多市場環境
- ★★ — 初步跡象，尚無統計支撐
- ★ — 效力存疑，請持續累積樣本

## 隨機基準（蒙地卡羅）

`compute_random_baseline(n_sim=1000)` 從 DB 隨機抽取同期股票，模擬 1000 次「隨機選股」的報酬分布。用於比較模型是否優於隨機。

## Beta 計算

`_calc_beta(ret_s, ret_m)` = Cov(個股報酬, 市場報酬) / Var(市場報酬)，採用 OLS。樣本 < 5 筆時回傳 `None`。

## 解讀限制

- 樣本數少時統計結果不穩定，信心評分自動降低
- 回測報酬不代表未來表現
- 未考慮滑價、流動性衝擊
